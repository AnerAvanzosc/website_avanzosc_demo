# Plan 1.0 — i18n base Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Activar ES + EU + EN en el sitio web por defecto y que las URLs `/es/…`, `/eu/…`, `/en/…` respondan 200 con el locale correcto. El switcher de idiomas del header de Odoo (aún sin re-estilar) aparece porque `website.language_ids` contiene >1.

**Architecture:** Toda la configuración vive como XML data records cargados al instalar/actualizar el módulo. `data/res_lang_data.xml` activa los 3 idiomas. `data/website_data.xml` establece `default_lang_id = es_ES` y `language_ids` en el website id=1. Un `post_init_hook` en Python invoca el wizard `base.language.install` para cargar las traducciones de Odoo estándar en cada idioma recién activado, ya que el mero `active=True` en `res.lang` no dispara la carga de PO files. Tests HttpCase verifican el estado resultante y smoke-testean las URLs con prefijo.

**Tech Stack:** Odoo 14 Community, Python 3.10, `res.lang`, `website`, `base.language.install` wizard, `odoo.tests.common.HttpCase`, XML data records con `noupdate`.

### Task 3: hooks.py — cargar traducciones estándar de Odoo

**Files:**
- Create: `hooks.py`
- Modify: `__init__.py`

Activar `res.lang` no instala las PO de Odoo. Sin eso, los strings del core saldrán en inglés aunque el usuario elija EU. Invocamos el wizard `base.language.install`.

- [ ] **Step 1: crear hooks.py**

`hooks.py`:
```python
# -*- coding: utf-8 -*-
from odoo import api, SUPERUSER_ID

TARGET_LANGS = ['es_ES', 'eu_ES', 'en_US']


def post_init_hook(cr, registry):
    """Instala las traducciones de Odoo estándar para los idiomas del sitio.

    res.lang.active=True por sí solo no carga las PO files. Usamos el wizard
    base.language.install que es la vía soportada por Odoo 14.
    """
    env = api.Environment(cr, SUPERUSER_ID, {})
    Lang = env['res.lang'].with_context(active_test=False)
    for code in TARGET_LANGS:
        lang = Lang.search([('code', '=', code)], limit=1)
        if not lang:
            continue
        if not lang.active:
            lang.active = True
        wizard = env['base.language.install'].create({'lang': code, 'overwrite': False})
        wizard.lang_install()
```

- [ ] **Step 2: re-exportar el hook desde `__init__.py`**

Editar `website_avanzosc_demo/__init__.py`:

```python

```

`data/website_data.xml`:
```xml
<?xml version="1.0" encoding="utf-8"?>
<odoo noupdate="1">
    <!-- Configura el website por defecto (id=1 del módulo `website`).
         noupdate=1 para NO pisar cambios posteriores del usuario en la UI
         (p.ej. si mañana añade fr_FR manualmente no queremos borrárselo). -->
    <record id="website.default_website" model="website">
        <field name="default_lang_id" ref="base.lang_es"/>
        <field name="language_ids" eval="[(6, 0, [
            ref('base.lang_es'),
            ref('base.lang_eu'),
            ref('base.lang_en'),
        ])]"/>
    </record>
</odoo>
```

Nota sobre los xmlids: el módulo `base` registra los idiomas como `base.lang_es` (es_ES), `base.lang_eu` (eu_ES) y `base.lang_en` (en_US). No son ni `lang_es_ES` ni nombres "obvios" — verificar con `odoo-bin shell` si hay dudas antes de confiar.

```python
    def test_url_prefixed_responses(self):
        """Frontend responde 200 para cada prefijo de idioma."""
        for code in ('es', 'eu', 'en'):
            r = self.url_open('/%s/' % code, timeout=15)
            self.assertEqual(
                r.status_code, 200,
                "GET /%s/ debe devolver 200 (fue %s)" % (code, r.status_code),
            )

    def test_root_serves_default_language(self):
        """GET / sirve contenido (Odoo puede redirigir al prefijo del default)."""
        r = self.url_open('/', timeout=15)
        self.assertIn(r.status_code, (200, 303),
                      "GET / debe ser 200 o 303 (redirect a /es/)")
```
